# BK6LS-Cadence — кастомная прошивка датчика каденса/скорости COOSPO BK6LS

[English](README.md) | **Русский**

![COOSPO BK6LS — стоковые режимы каденса и скорости](image.png)

Полностью открытая кастомная прошивка для BLE-датчика каденса/скорости
**COOSPO BK6LS** (BK6LS-0026006) на SoC **nRF52810**. Стоковая прошивка была
отреверс-инжинирена (распиновка, акселерометр, светодиоды — см.
[HARDWARE.md](HARDWARE.md)) и заменена приложением на Zephyr, которое сохраняет
совместимость с обычными велоприложениями **и** добавляет живой поток RPM со
знаком (направлением вращения) для контроллеров электровелосипедов.

> На картинке выше — стоковое переключение режима переустановкой батарейки
> (каденс = синий, скорость = красный). Текущая прошивка работает в режиме
> **каденса (шатун)**; переключение плоскости для режима колеса — в планах.

## Возможности

- **Стандартный BLE-профиль Cycling Speed & Cadence (`0x1816`)** — из коробки
  работает с Wahoo/Garmin/любым велоприложением.
- **Кастомный RPM-сервис** — знаковый `int16` в санти-RPM, уведомления каждые
  **100 мс**, знак = направление вращения (вперёд/назад). UUID
  `cad00001-eb1c-4f1e-9b2a-6f1c0de0cade`. Полное руководство для клиента:
  [BLE_INTERFACE.md](BLE_INTERFACE.md).
- **Battery Service (`0x180F`)** — проценты по напряжению «таблетки» через
  внутренний SAADC (без внешних пинов).
- **Магниты не нужны** — каденс вычисляется по **вращающемуся вектору
  гравитации** встроенного 3-осевого акселерометра (θ = atan2 осей в плоскости
  вращения; полный оборот = один оборот шатуна, RPM = dθ/dt).
- **Адаптивное энергопотребление** — ACTIVE: сэмплирование 100 Гц + advertising;
  после 20 с без движения и без подключения advertising выключается, устройство
  переходит в низкопотребляющий опрос раз в 5 с; любое изменение показаний
  акселерометра будит его обратно.
- **Индикация** — синий 1 с = пробуждение/включение, красный 1 с = засыпание
  (один двухцветный светодиод, активный уровень — низкий).

## SDK / инструменты

| Компонент | Версия |
|---|---|
| **nRF Connect SDK (NCS)** | **v3.2.4** |
| RTOS | Zephyr (в составе NCS) |
| BLE-контроллер | Nordic SoftDevice Controller, только peripheral (`CONFIG_BT_LL_SOFTDEVICE`) |
| Цель сборки | `nrf52dk/nrf52810` + [`app.overlay`](app.overlay) |
| Прошивка | `nrfjprog` + J-Link (SWD) |

Чип — ограниченный nRF52810 (24 КБ RAM / 192 КБ flash), поэтому в
[`prj.conf`](prj.conf) урезаны BLE-буферы и лишние фичи.

## Сборка

Сборка выполняется в окружении тулчейна NCS v3.2.4 (Windows; см.
[`scripts/ncs-env.ps1`](scripts/ncs-env.ps1)):

```powershell
. .\scripts\ncs-env.ps1
west build -b nrf52dk/nrf52810 .
```

## Прошивка

```powershell
nrfjprog --family NRF52 --program release\bk6ls-cadence-<version>.hex --chiperase --verify --reset
```

Заводская прошивка сохранена в
[`stock-backup/stock_full.hex`](stock-backup/stock_full.hex) — её можно
восстановить той же командой в любой момент.

## Релиз

Готовые образы прошивки лежат в [`release/`](release/). Как собрать релиз:

```powershell
.\scripts\make-release.ps1              # собирает и кладёт hex в release/
.\scripts\make-release.ps1 -Version 1.0.0
.\scripts\make-release.ps1 -NoBuild     # упаковать уже собранный build/
```

Скрипт делает чистую сборку `west build`, копирует hex в
`release/bk6ls-cadence-<version>.hex` и пишет рядом `.txt` с коммитом, датой и
SHA-256.

## Структура репозитория

```
src/            прошивка: главный цикл, математика каденса, драйвер акселерометра, GATT (CSC/RPM/батарея)
tools/pinscan/  bring-up-прошивка для реверса распиновки и идентификации акселерометра
tools/idle/     «пустая» прошивка, чтобы дёргать пины из отладчика
scripts/        окружение NCS + скрипт релиза
stock-backup/   полный дамп заводской прошивки (восстанавливаемый)
release/        готовые релизные hex-файлы
HARDWARE.md     заметки по реверс-инжинирингу (распиновка, регистры акселерометра, светодиоды)
BLE_INTERFACE.md руководство по интеграции для BLE-клиента (самодостаточное)
```

## Статус / планы

- [x] CSC + кастомный знаковый RPM + батарея, адаптивный сон/пробуждение
- [ ] Переключение плоскости шатун/колесо (стоковая «смена режима переустановкой батарейки»)
- [ ] Аппаратное пробуждение по движению через INT-пин акселерометра (P0.15/P0.25) + System OFF

## Дисклеймер

Проект — результат реверс-инжиниринга серийного устройства. Прошивка кастомного
ПО лишает гарантии и может «окирпичить» датчик. Используйте на свой страх и
риск. Проект не связан с COOSPO.
